What does this site do?

If you click the button at the bottom of the page, there will be an iframe shown.

There were two service workers registered when you've opened this file. One of them is related to this "main" frame, while the other is under the path of the iframe.

What does it want to investigate?

How can the main frame know if either its own or its child frame's service worker was updated.

How to test?

  1. Load the page, and check that both service workers were properly registered (you might need to tweak either the certs used or your browser
  2. Change something in the iframe's package (`packages/child/src/index.js`)
  3. Reload the page, and check that there is a new version for the main frame's service worker
  4. Click the button to load the iframe
  5. Check that there is a new version for the iframe's service worker

What was expected?

  • When the page is reloaded, there is no new version for the main frame's service_worker, instead
  • there is a new version for the iframe's service worker without loading the iframe
